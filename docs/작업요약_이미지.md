## 0. 한 줄 요약

인계 문서가 "통째로 없다"고 한 **이미지 추출① 단계**를 신규 구현하고, 생성②·안전검사까지
`app/image/` 모델링 패키지로 완성했다. 핵심 재설계: **표지와 관계도는 별개 플로우**로 분리했다.

---

## 1. 가장 큰 설계 결정 — 표지 / 관계도 분리

처음엔 추출①을 캐릭터·관계를 한 번에 뽑는 단일 모듈로 잡았으나, 요구사항 검토 결과
표지와 관계도는 요구사항 ID·엔드포인트·테스트·CRUD가 모두 분리된 **독립 플로우**임을 확인하고
플로우별로 추출을 분리했다.

| | 표지 플로우 | 관계도 플로우 |
| --- | --- | --- |
| 요구사항 | REQ-IMG-001/002/003 | REQ-IMG-004/005/006 |
| 테스트 | TST-IMG-001 | TST-IMG-002 |
| 엔드포인트 | `/api/generate-cover-image` | `/api/generate-relation-image` |
| 추출 분량 | 최대 **10화** | 최대 **20화** (관계는 누적할수록 풍부) |
| 추출 내용 | 캐릭터 외형 **+ 해당 분량 행보/임팩트** | 인물 노드 + 관계(방향성 포함) |
| 안전검사 | **있음** (인물=신체 묘사 → 노출 위험) | 없음 (다이어그램) |
| 추출 정보 수명 | 표지 생성 후 휘발 | 관계도 생성 후 휘발 |

표지 캐릭터에는 외형뿐 아니라 `arc_summary`(이 분량에서의 행보)와 `key_moments`(임팩트 장면)를
넣어, 단순 인물 일러스트가 아니라 그 화차의 서사를 담은 표지가 되도록 했다.

---

## 2. 확정한 설계 결정 (인계 문서 5번 미해결 항목)

1. **추출 입력 단위**: 다중 화 누적. 리스트로 받고 1개만 줘도 동작. (표지 10화 / 관계도 20화 상한)
2. **추출 출력 형식**: 전체 필드.
   - 표지 캐릭터: `name, gender, age_estimate, appearance[], personality, role, arc_summary, key_moments[]`
   - 관계도: 노드 `{name, role}` + 관계 `{from, to, relation_type, directed, evidence}`
   - `directed`: 짝사랑 등 한쪽 방향(true) vs 대칭(false) → 관계도 화살표 방향(→ / ↔)
   - `evidence`: 그 관계로 판단한 한국어 원문 근거 한 줄 (검증·환각 억제용, 번역 inspector 패턴과 일관)
3. **표지 인물 수**: 주인공 1명 기본. 사용자가 추가 프롬프트로 조정 가능.
4. **추출 RAG**: 순수 LLM(원문만). `config.use_rag` 스위치만 남겨 추후 kculture RAG 확장 여지 확보.
5. **역할 분담**: `app/image` = 추출+생성+안전검사 모델링, `backend/services/image_service` = 호출+HTTP 포장.

## 3. 만든 파일 (`app/image/`)

translation 과 동일하게 역할별 하위 폴더로 구조화했다.

```
app/image/
├── __init__.py              공개 API export
├── config.py                ImageConfig (모델명, 화 상한 10/20, use_rag·safety 스위치)
├── cover_pipeline.py        [표지] CoverPipeline: episodes → 추출 → 생성 → CoverResult
├── relation_pipeline.py     [관계도] RelationPipeline: episodes → 추출 → 생성 → RelationResult
├── core/                    공통 인프라
│   ├── _extract_base.py     join_episodes(화 제한만) + call_structured(Responses API+json_schema strict)
│   ├── _generate.py         generate_image 공통 (mock 시 type=mock_image)
│   └── mock_adapters.py     cover/relation 추출 mock (결정적 응답)
├── extract/                 ① 추출
│   ├── cover_extractor.py   CoverExtractor → CoverCharacter(외형+arc_summary+key_moments)
│   └── relation_extractor.py RelationExtractor → RelationNode + Relation(directed 포함)
└── generate/                ② 생성
    ├── cover_generator.py   CoverGenerator (프롬프트 조립 + 안전검사 → 이미지)
    ├── relation_generator.py RelationGenerator (→/↔ 방향 반영한 관계도 프롬프트)
    └── safety.py            is_unsafe_visual_request(사전필터) + build_refusal(LLM 거부+대안)
```

**파이프라인(오케스트레이터)**: 번역의 `TranslationPipeline` 패턴을 그대로 따른다.
`CoverPipeline.run(episodes, ...)` → `CoverResult{extraction, image}` (추출+생성 묶음, `refused` 프로퍼티).
이미 추출된 결과로 생성만 하려면 `generate_from_extraction()`.

설계 일관성: 번역 엔진과 동일하게 `get_openai_client()`, `is_mock_mode()`(`WLIGHTER_MOCK_MODE`),
Responses API + `json_schema`(strict), dataclass 기반 config·결과 구조를 재사용했다.

### 안전검사 (거부 + 대안)
인계 문서 요구대로 "단순 거부"가 아니라 "거부 + 대안 제시"로 보강했다.
사전 키워드 필터로 1차 차단 후, LLM이 "어떤 부분이 불가한지 + 이렇게 조정하면 가능"을 맥락에 맞게 생성한다.
mock 모드/LLM 실패 시 `생성해드릴 수 없습니다`를 포함한 고정 폴백 메시지로 떨어진다.

## 4. backend/services/image_service.py 전환

기존 생성 로직을 `app/image`로 이전하고, 서비스는 **얇은 포장**으로 바꿨다.
`cover_image(payload)` / `relation_image(payload)`는:
- payload에 `episodes`가 있으면 → 신규 추출 플로우(원문 → 추출 → 생성)
- 없으면 → 레거시 payload(`protagonist`/`appearance`/`characters`/`relations`)를 추출 결과로 감싸 생성

덕분에 기존 `api_server.py`·테스트·`model_test_case_runner.py` 호환성을 그대로 유지한다.
출력 계약(`type=="mock_image"`, `notice`, 프롬프트 내 `vertical commercial web novel cover` /
`title-safe negative space` / `relationship map`)도 보존했다.

## 7. 샌드박스 제약 (중요)

- 개발 샌드박스 프록시가 `api.openai.com`을 **403 차단**한다 → `mock=False` 실제 LLM/이미지 호출은
  샌드박스에서 불가. **로컬에서 실행해야 한다** (`.env`에 `OPENAI_API_KEY` 존재 확인됨).
- 로컬 실행 시 패키지: `pip install openai "httpx[socks]" socksio` (SOCKS 프록시 환경).

```bash
# mock 검증 (네트워크 불필요)
WLIGHTER_MOCK_MODE=true python tests/check_image_extraction.py
WLIGHTER_MOCK_MODE=true python -m pytest tests/test_image_extraction.py -q

# 실제 추출/생성 (로컬, OPENAI_API_KEY 필요)
python tests/check_image_extraction.py            # 추출까지
python tests/check_image_extraction.py --images   # 이미지 생성까지 (비용 발생)
```
